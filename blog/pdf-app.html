Developing WizPDF: My First Mobile App for iOS and Android

April 3, 2024 • Mobile Development, iOS, Android, PDF Tools

In an age where digital documents are ubiquitous, I found myself constantly wrestling with PDFs. Merging research papers, extracting specific pages, and needing quick summaries—these seemingly simple tasks often required jumping between multiple desktop apps or expensive subscriptions. Frustrated by this workflow, I decided to build WizPDF, a sleek mobile app that puts professional PDF tools right in your pocket.

Why I Built This App

As someone who regularly handles dozens of PDF documents, I kept hitting the same pain points:

  • Needing to combine multiple research papers into a single document
  • Wanting to rearrange or remove specific pages from large PDFs
  • Requiring quick summaries of lengthy documents
  • Doing all this on-the-go, without a desktop computer

Most mobile PDF solutions were either too limited, ridiculously expensive, or buried these functions under confusing interfaces. I wanted to create something that was both powerful and delightfully simple to use—and WizPDF was born.

Core Features

PDF Merging

Select and combine multiple PDFs into a single document with a few taps. Easily rearrange the order of files before merging, with a clean interface that makes the process intuitive and fast.

Page Manipulation

Reorder, delete, and extract pages from PDF documents with our visual page interface. Preview each page thumbnail and easily rearrange with intuitive up/down controls.

PDF Summarization

Generate AI-powered summaries of PDF documents in short, medium, or long formats. Perfect for quickly extracting key information from lengthy papers or reports.

PDF Processing Engine

Working with PDFs on mobile devices presented several challenges, particularly around memory management and performance. I used the powerful pdf-lib library for core operations like merging and page manipulation: 


export const manipulatePDF = async (options: ManipulateOptions): Promise<string> => {
  try {
    const pdfBytes = await FileSystem.readAsStringAsync(options.file.uri, {
      encoding: FileSystem.EncodingType.Base64,
    });
    
    const pdf = await PDFDocument.load(pdfBytes);
    const newPdf = await PDFDocument.create();
    
    // Copy pages in the specified order
    for (const pageIndex of options.pageOrder) {
      const adjustedIndex = pageIndex - 1;
      if (adjustedIndex >= 0 && adjustedIndex < pdf.getPageCount()) {
        const [copiedPage] = await newPdf.copyPages(pdf, [adjustedIndex]);
        newPdf.addPage(copiedPage);
      }
    }
    // Save and return the new document...
  }
};

This approach allowed for efficient PDF manipulation right on the device, without sending sensitive documents to external servers.

Challenges and Solutions

Performance Optimization

Handling large PDF files on mobile devices required careful optimization to prevent crashes and maintain smooth performance:

  • Implemented efficient file loading and processing for large documents
  • Optimized memory usage when handling multiple PDFs
  • Added loading indicators to provide feedback during processing

Cross-Platform File Handling

File system access varies significantly between iOS and Android, requiring platform-specific implementations: 


export const sharePDF = async (filePath: string): Promise<void> => {
  try {
    if (Platform.OS === 'android') {
      // Android-specific file system access
      const permissions = await FileSystem.StorageAccessFramework.requestDirectoryPermissionsAsync();
      // Handle Android storage...
    } else {
      // iOS sharing is simpler
      const canShare = await Sharing.isAvailableAsync();
      if (canShare) {
        await Sharing.shareAsync(filePath);
      }
    }
  } catch (error) {
    console.error('Error sharing PDF:', error);
    throw error;
  }
};

App Store Approval Process

Navigating the App Store approval process was an education in itself. Apple's guidelines for in-app purchases are particularly strict, requiring careful implementation of purchase restoration and clear disclosure of pricing.

Conclusion

Building WizPDF has been an incredibly rewarding project, from identifying a personal need to creating a polished product that solves real problems. The combination of React Native with Expo, Firebase Cloud Functions, and pdf-lib has enabled me to create a powerful yet intuitive PDF toolkit that lives right on your mobile device.

I hope this behind-the-scenes look at WizPDF's development provides some insight for other developers looking to create their own mobile apps. If you're interested in trying WizPDF yourself, it's available now on the App Store and Google Play.
